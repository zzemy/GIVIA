# Homepage Final Polish Design

## Summary

This final polish pass keeps the upgraded Luxury Advisor System homepage structure, but pushes it from "good redesign" to "finished product" quality. The focus is no longer large-scale restructuring. Instead, it is about rhythm, visual emphasis, mobile stacking quality, and the removal of the last remaining traces of dashboard-style noise.

## Scope

This pass covers:
- hero rhythm and first-screen emphasis
- workflow stage hierarchy and card density
- secondary result-panel noise reduction
- mobile stacking and spacing cleanup
- final visual-system consistency for borders, shadows, contrast, and spacing

This pass does not change backend logic, API contracts, locale scope, or the five-card insight concept.

## Approved direction

### 1. Hero and page rhythm
- Make the hero feel calmer and more deliberate without changing its structure.
- Increase the distinction between primary message, supporting explanation, CTA row, and trust/supporting metadata.
- Reduce the visual loudness of nonessential elements such as secondary chips and supporting blocks.
- Normalize vertical spacing between the major page bands: hero, insights, workflow, and results.

### 2. Workflow hierarchy
- Keep the workflow as the clear primary task stage.
- Push assistant/history/support surfaces further into secondary territory.
- Make the analysis card read as a decision step first and a configuration surface second.
- Keep advanced controls available, but never let them dominate the page.

### 3. Results hierarchy
- Keep recommendation-first reading order exactly as implemented.
- Further reduce the prominence of secondary data panels so the advisor summary remains the anchor.
- Preserve professional depth while making the detail layer feel quieter, cleaner, and easier to scan.

### 4. Mobile polish
- Make mobile read as a clean sequence instead of a compressed stack of similarly weighted cards.
- Ensure CTA order and spacing remain clear on smaller screens.
- Ensure workflow and results preserve the same hierarchy as desktop rather than merely shrinking the same density.

### 5. Visual system cleanup
- Unify border contrast and shadow intensity across premium, secondary, and quiet surfaces.
- Align typography contrast so headings, body, labels, and metadata always read in the intended order.
- Remove any remaining mixed-style areas that still feel inherited from the older dashboard phase.

## Success criteria

1. The hero feels more premium because of spacing and rhythm, not because of extra decoration.
2. The workflow reads as one guided path, not a cluster of equal cards.
3. The result section still leads with the advisor summary, with lower visual noise behind it.
4. Mobile feels intentionally designed rather than responsively compressed.
5. Borders, shadows, and text contrast feel like one coherent system across the full homepage.
